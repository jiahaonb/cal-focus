# Cal Focus — 技术设计与交互异常场景说明（架构设计 v1.0）

## 1. 文档目的

本文件用于衔接 PRD 与编码实施，重点回答：

1. 采用什么工程架构实现“主程序 + 托盘 + 多桌面小组件”。
2. 关键交互流程如何设计，保证体验流畅且易理解。
3. 多显示器、DPI、异常中断、性能退化等复杂场景如何处理。
4. 后续编码阶段如何按模块推进并持续可维护。

---

## 2. 总体架构

## 2.1 技术栈（落地）

- 语言：`C#`（.NET 8）
- UI：`WinUI 3`（Windows App SDK）
- 系统材质/动画：`System Backdrop + Windows Composition`
- 数据：`SQLite`（业务数据）+ `JSON`（UI配置/主题/布局）
- 通知：Windows Toast

## 2.2 进程与窗口模型

单进程多窗口：

1. **主窗口（MainWindow）**：设置中心与日历/计划操作中心。
2. **小组件窗口（WidgetWindow[]）**：每个小组件实例独立窗口。
3. **托盘控制器（TrayController）**：全局入口，不依赖主窗口可见性。

设计原则：
- 主窗口可关闭到托盘（不退出进程）。
- 小组件窗口生命周期由 `WidgetHostService` 管理。
- 退出应用时统一销毁所有小组件窗口并持久化状态。

## 3. 分层与模块边界

## 3.1 目录建议

```text
src/
  CalFocus.App/                 # 启动与依赖注入
  CalFocus.UI/                  # WinUI 页面与控件
  CalFocus.Core/                # 业务实体、用例、规则
  CalFocus.Infrastructure/      # SQLite、文件、系统 API 适配
  CalFocus.Widgets/             # 小组件窗口与渲染
  CalFocus.Theming/             # 主题、材质、视觉 token
```

## 3.2 核心模块职责

1. **TrayController**
   - 创建托盘图标
   - 响应左键/右键菜单
   - 转发命令到 `AppCommandBus`

2. **DisplayService**
   - 检测显示器列表、DPI、边界、主副屏
   - 监听显示器拓扑变化（插拔、分辨率变化）

3. **WidgetHostService**
   - 创建/销毁小组件窗口
   - 处理跨屏拖拽、吸附、层级
   - 负责布局持久化与恢复

4. **Schedule/Reminder/Plan Services**
   - CRUD + 规则计算（重复任务、提醒触发）

5. **ThemeEngine**
   - 材质模式切换：Glass / Flat / Depth
   - 背景图、透明度、模糊强度参数应用

6. **Persistence Layer**
   - SQLite：任务、日程、计划、提醒
   - JSON：窗口布局、主题参数、用户偏好

## 4. 关键数据模型（建议）

## 4.1 业务模型

- `ScheduleItem`
  - `Id`, `Title`, `StartAt`, `EndAt`, `RepeatRule`, `Category`, `Color`, `Note`
- `ReminderItem`
  - `Id`, `Title`, `TriggerAt`, `RepeatRule`, `SnoozePolicy`, `SoundEnabled`
- `PlanItem`
  - `Id`, `Title`, `PeriodType(Week/Month)`, `Status`, `RelatedScheduleIds`

## 4.2 UI与布局模型

- `WidgetInstance`
  - `Id`, `WidgetType`, `DisplayId`, `X`, `Y`, `Width`, `Height`, `Opacity`, `StylePreset`, `Locked`

- `ThemeProfile`
  - `Mode(Glass/Flat/Depth)`, `LightDark`, `BackgroundImagePath`, `BackgroundOpacity`, `BlurStrength`, `TintStrength`, `NoiseStrength`

## 5. 交互体验设计（详细）

## 5.1 首次启动引导

1. 欢迎页：说明“托盘常驻 + 小组件可拖动到任意屏幕”。
2. 权限与系统建议：是否开机启动、是否启用通知。
3. 屏幕识别页：显示所有屏幕缩略图。
4. 快速添加：默认添加 1 个时钟组件到主屏。

目标：用户 30 秒内看到“可用成果”，降低学习成本。

## 5.2 托盘交互

- 左键：显示/激活主窗口（若已打开则置顶）
- 右键菜单：
  - 打开主界面
  - 添加小组件
  - 显示/隐藏全部小组件
  - 开机启动（勾选）
  - 退出

体验细节：
- 右键菜单应在 100ms 内弹出。
- 菜单项应体现当前状态（例如“已隐藏全部小组件”时给出恢复入口）。

## 5.3 主窗口导航交互

- 左侧导航保持固定顺序，避免频繁变动。
- 页面切换动画统一 200ms 左右，避免“晕动感”。
- 顶部提供全局“快速添加”按钮（添加日程/提醒/小组件）。

## 5.4 小组件添加流程

1. 进入“小组件中心”。
2. 选择组件类型与外观模板。
3. 选择目标屏幕（可预览屏幕编号）。
4. 点击“添加到桌面”。
5. 新组件出现并进入可拖拽状态。

失败反馈：
- 若显示器信息读取失败：提示“请检查系统显示设置后重试”。
- 若创建窗口失败：给出“重试”按钮并记录日志。

## 5.5 拖拽与跨屏体验

- 拖拽时显示半透明预览框。
- 接近屏幕边缘时出现目标屏幕高亮。
- 释放后自动吸附到最近网格（可关闭）。
- 若组件被锁定：拖拽前需二次确认“解锁后移动”。

## 5.6 主题与外观体验

- 用户可以实时预览材质变化（不需要离开设置页）。
- 背景图更换后立即预览并可撤销。
- “平面/立体/毛玻璃”切换要明确风格差异说明。

## 6. 异常场景与处理策略（重点）

## 6.1 多显示器变化

### 场景 A：副屏断开
- 问题：该屏上的组件不可见。
- 处理：自动迁移到主屏安全区域（右下角堆叠），并弹出一次轻提示。
- 数据：保留原 `DisplayId`，副屏恢复后允许“一键回到原位置”。

### 场景 B：主副屏切换
- 问题：工作区坐标变化导致组件偏移。
- 处理：按 `WorkArea + DpiScale` 重新计算组件坐标，超出区域则回调边界内。

### 场景 C：DPI 动态变化（100% -> 150%）
- 问题：组件尺寸视觉失真。
- 处理：基于设备无关像素保存布局，渲染时按当前 DPI 转换。

## 6.2 托盘/进程状态异常

### 场景 D：主窗口误关闭
- 处理：默认最小化到托盘，不退出进程；首次触发弹出引导提示。

### 场景 E：托盘图标丢失（Explorer 重启）
- 处理：监听任务栏重建消息并自动重建图标。

## 6.3 数据异常

### 场景 F：配置文件损坏
- 处理：
  1. 备份损坏文件到 `corrupted/`。
  2. 使用默认配置恢复。
  3. 提示用户可从最近备份恢复。

### 场景 G：SQLite 读写失败
- 处理：
  1. UI 提示“数据暂不可写，已切只读模式”。
  2. 保证小组件继续显示最后可用数据。
  3. 日志记录错误并提示重启修复。

## 6.4 视觉兼容异常

### 场景 H：显卡性能不足导致毛玻璃卡顿
- 处理：自动降级到低质量（降低模糊半径、关闭噪点层）。
- 用户可手动固定质量档位，防止频繁切换。

### 场景 I：Windows 10 不支持完整材质
- 处理：降级为半透明 + 模拟模糊背景，保留设计语言一致性。

## 6.5 交互冲突

### 场景 J：点击穿透开启后无法选中组件
- 处理：托盘菜单提供“临时关闭穿透（30 秒）”快速入口。

### 场景 K：组件重叠无法操作
- 处理：提供“层级整理”功能（置顶、下移、自动平铺）。

## 7. 性能与体验保障

## 7.1 性能预算

- 空闲 CPU：目标 < 2%
- 内存基础占用：目标 < 220MB
- 小组件拖动：目标 60FPS
- 新增一个组件响应：目标 < 300ms

## 7.2 优化策略

- 小组件刷新节流（例如 1 秒一次）
- 非可见屏幕的组件降频渲染
- 复杂动画在低电量模式自动简化
- 大图背景加载时使用异步解码与缓存

---

## 8. 可观测性与排障

- 关键日志分类：`Tray`, `Display`, `Widget`, `Theme`, `Data`
- 每次异常附带：错误码、模块、屏幕拓扑、DPI、最近操作
- 提供“导出诊断包”功能（日志 + 配置快照，不含隐私内容）

---

## 9. 安全与隐私

- 默认本地存储，不上传云端
- 背景图路径与个人文件路径仅本地保存
- 日志默认不记录日程正文（可选开启调试模式）

## 12. 近期实现后的改进建议（新增）

基于当前已落地代码（托盘、组件管理、主页三列布局）建议优先补齐：

1. **开机启动兼容性增强**
   - 现状：已支持通过注册表 `Run` 项开关开机启动。
   - 建议：后续增加“权限/写入失败”的 UI 提示，避免用户误判已开启。

2. **小组件布局持久化（已完成 MVP）**
   - 现状：已实现添加/删除/复制/拖拽尺寸变化后的自动保存（500ms 防抖）与启动恢复。
   - 建议：下一步补齐“按屏幕恢复（DisplayId）”与“屏幕断连回退策略”。

3. **小组件复制能力增强**
   - 现状：已支持复制最后一个组件（偏移 28px）。
   - 建议：下一步增加“复制指定组件（按 Id）”与“复制后自动聚焦”，方便多实例编辑。

4. **主页三列数据接口化**
   - 现状：天气/计划/提醒/日程为静态占位数据。
   - 建议：统一抽象为 `DashboardSnapshot`，后续对接数据库与天气 API 时仅替换数据源，不改布局层。

5. **托盘状态反馈优化**
   - 现状：已支持显示/隐藏全部小组件、开机启动切换。
   - 建议：托盘菜单文案动态化（如“全部已隐藏/全部已显示”），提升状态可见性。

6. **多屏布局模型升级（持续推进）**
   - 现状：已完成基础显示器枚举、主屏识别、坐标命中与 `DisplayId` 基础写入恢复。
   - 建议：补齐屏幕断连重连后的自动回迁与跨屏拖拽吸附策略。

7. **F5 自动回迁基础版（已落地）**
   - 现状：已实现显示拓扑轮询检测（3 秒）与组件自动回迁到可用屏幕工作区。
   - 建议：下一步替换为事件驱动拓扑监听，并增加“回迁提示”与“一键回原屏”能力。
