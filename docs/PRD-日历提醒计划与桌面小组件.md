# Cal Focus — 软件需求功能文档（PRD v1.0）

## 1. 项目目标

打造一款以 **日历日程 + 提醒 + 计划管理 + 桌面小组件** 为核心的一体化 Windows 桌面软件。

产品定位：
- 视觉上：接近 iOS 26 风格的简约大气 + 高级毛玻璃质感
- 功能上：从托盘快速进入，完整管理任务与日程，并可将时钟/日程等小组件投放到多显示器桌面
- 体验上：轻量、流畅、低打扰，强调“随时看见、随时安排”

---

## 2. 目标用户

- 重度日程管理用户（工作/学习排期）
- 多显示器办公用户（希望在不同屏幕放不同信息）
- 对桌面美感与 UI 质感要求高的用户

---

## 3. 关键需求概述

1. **任务栏托盘入口**
   - 常驻托盘图标
   - 左键：打开主窗口
   - 右键：菜单（打开设置、显示/隐藏小组件、开机启动、退出）

2. **主窗口（设置与管理中心）**
   - 左侧导航栏切换模块：
     - 日历日程
     - 提醒事项
     - 计划表（周/月计划）
     - 小组件中心
     - 主题与外观
     - 系统设置

3. **桌面小组件系统**
   - 支持：桌面时钟、桌面日程（后续可扩展倒计时、习惯追踪等）
   - 可从“小组件中心”一键添加到指定显示器
   - 支持拖动移动，跨屏拖动
   - 支持锁定位置、置顶层级、点击穿透（可选）

4. **多显示器能力**
   - 自动检测所有显示器（分辨率、缩放比例、主副屏）
   - 配置界面中可预览每个屏幕并投放小组件
   - 每块屏幕可保存独立布局

5. **视觉风格（最重要）**
   - iOS 风格“毛玻璃”主题（Glass）
   - 支持“平面模式 / 立体模式”切换
   - 背景图可自定义，且可调不透明度

---

## 4. 技术选型建议（结论）

> 首选技术栈：**C# + WinUI 3 + Windows App SDK + Windows Composition**

### 4.1 选择理由

- 与 Windows 系统材质能力（Mica/Acrylic）兼容性最佳
- 对高质量桌面 UI、动画、材质分层支持更强
- 便于实现多窗口（主窗口 + 多个小组件窗口）与系统集成（托盘、开机启动、多屏）
- 性能与内存可控，优于基于浏览器内核的常驻方案

### 4.2 联网调研要点（用于实现“iOS 质感”）

结合 Windows 官方材质能力：
- **Mica**：适合主应用背景，稳定、低功耗
- **Desktop Acrylic**：更强的磨砂/半透明视觉，适合小组件卡片层
- 搭配 Composition 做高光、阴影、边缘描边、动态模糊，构建接近 iOS 的“液态玻璃感”

### 4.3 备选方案（不优先）

- Electron/Tauri：开发快，但常驻内存与系统一致性一般，不作为首选
- WPF：成熟但在新式系统材质与动画表现上弱于 WinUI 3

## 5. 信息架构与功能模块

## 5.1 日历日程
- 月/周/日视图
- 点击日期查看当日日程
- 支持新建、编辑、删除日程
- 支持重复规则（每天/每周/每月）
- 支持颜色分类（工作/生活/学习）

## 5.2 提醒事项
- 快速提醒（一次性）
- 循环提醒（按周期）
- 提醒方式：系统通知 + 声音（可选）
- 到期未处理可稍后提醒

## 5.3 计划表
- 周计划 / 月计划看板
- 计划与日程可关联
- 进度状态：未开始 / 进行中 / 已完成

## 5.4 小组件中心
- 小组件列表（时钟、今日日程、本周计划）
- 样式模板选择（平面 / 立体 / 玻璃）
- 添加时选择目标显示器
- 每个小组件支持尺寸、透明度、圆角、阴影、描边调整

## 5.5 主题与外观
- 主题：浅色/深色/自动
- 风格：
  - Glass（毛玻璃）
  - Flat（平面）
  - Depth（立体）
- 自定义背景图（主窗口与小组件可分开设置）
- 背景不透明度滑杆（0–100%）
- 玻璃参数：模糊强度、染色强度、噪点强度（高级项）

## 5.6 系统设置
- 托盘行为设置
- 开机自启动
- 数据备份与恢复（本地）
- 多显示器布局重置

---

## 6. UI / 视觉规范（iOS 26 气质）

### 6.1 总体风格关键词
- 清透、轻盈、克制
- 强层次但低噪音
- 动效柔和、过渡自然

### 6.2 毛玻璃实现分层（建议）

1. **底层**：壁纸/自定义背景图（可调 opacity）
2. **材质层**：Mica/Acrylic（按窗口类型选择）
3. **色彩染层**：冷色轻 tint（可主题化）
4. **细节层**：
   - 1px 半透明高光描边
   - 软阴影（低扩散）
   - 轻噪点纹理（避免塑料感）

### 6.3 平面 / 立体模式差异
- 平面：低阴影、低高光、低景深
- 立体：增强阴影、边缘高光、轻透视位移、悬浮反馈

### 6.4 动效规范
- 基础过渡：180–260ms
- 弹性进入：220–320ms
- 鼠标悬停微位移 + 高光变化
- 拖动小组件时显示吸附参考线与目标屏幕提示

---

## 7. 多显示器与小组件交互要求

1. 启动时扫描显示器并建立屏幕模型（ID、Bounds、DPI）
2. 小组件创建时必须绑定初始屏幕
3. 拖拽越过屏幕边界时自动切换目标屏幕
4. 屏幕断连后：
   - 小组件临时回收到主屏安全区域
   - 记录原屏幕 ID，恢复后可还原
5. 支持每屏独立布局导入/导出

---

## 8. 数据与配置

## 8.1 数据对象
- Schedule（日程）
- Reminder（提醒）
- Plan（计划）
- WidgetInstance（小组件实例）
- DisplayProfile（显示器配置）
- ThemeProfile（主题配置）

## 8.2 本地存储
- 建议 SQLite + JSON 配置
- 配置拆分：
  - `appsettings.json`（全局）
  - `widgets.layout.json`（布局）
  - `theme.profile.json`（主题）

---

## 9. 非功能需求

- 冷启动 < 2.5s（普通办公本）
- 托盘常驻内存控制：基础态建议 < 220MB
- 小组件拖动帧率稳定（目标 60 FPS）
- 崩溃自动恢复最近布局
- 支持 Windows 11（首发），Windows 10（降级视觉兼容可选）

## 11. 验收标准（首版）

- 可以通过托盘完整管理和打开主程序
- 日程、提醒、计划三模块可用且数据持久化
- 至少两种小组件可添加到任意显示器
- 小组件支持跨屏拖动并保存位置
- 主题支持 iOS 风格毛玻璃，并可切换平面/立体
- 背景图可自定义且不透明度可调

---

## 12. 风险与对策

1. **风险：毛玻璃效果在不同显卡表现差异**
   - 对策：提供质量档位（高/中/低）
2. **风险：多显示器 + DPI 导致坐标偏移**
   - 对策：统一 DPI 感知与屏幕坐标转换层
3. **风险：小组件过多导致性能下降**
   - 对策：限制刷新频率、可见区优先渲染
